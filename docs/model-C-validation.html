Model C — Structure-Conditioned Generation

The proposed replacement for the MediaPipe-as-loss design: feed an edge / silhouette map as a per-image condition. This page is mostly about one thing — how to prove the idea works in under an hour, before you commit to a full training run.

The idea in one line

Instead of scoring generated hands with landmarks (which fails at 27% detection and regresses to the mean), give the model the hand structure as input and ask it to render the photo. The structure map and the target image come from the same image → real correspondence.

real image→ edge / silhouette map+ class label→ Generator→ rendered hand

Discriminator sees the pair (image, condition) — so it enforces "this image matches this structure & label". This is the standard pix2pix setup; the same condition later drives a ControlNet-style diffusion model.

The condition is computed for 100% of images with no model and no failures:

import cv2, numpy as np
def structure_map(gray_u8):
    edge = cv2.Canny(gray_u8, 60, 160)              # contour of the hand
    _, sil = cv2.threshold(gray_u8, 0, 255,
                           cv2.THRESH_BINARY+cv2.THRESH_OTSU)  # silhouette
    dist = cv2.distanceTransform(sil, cv2.DIST_L2, 3)     # smooth shape cue
    return np.stack([edge, sil, cv2.normalize(dist,None,0,255,cv2.NORM_MINMAX)], -1)

Build steps (high level)

1

Compute conditions

Run structure_map() over all 54k images → cache as a parallel array (like the landmark cache, but 100% coverage).

edge + silhouette + distance
2

Reuse the optimized backbone

Keep the SAGAN generator / spectral-norm discriminator from src/models.py. Change the generator input to accept the condition map; change D to take (image, condition, label).

3

Pair condition ↔ target

The training target for condition C(x) is image x itself. This single change removes the regress-to-mean problem.

4

Loss = adversarial + paired L1

Now the L1 is aligned (fake vs the exact image the condition came from), so it sharpens instead of blurring.

G = g_adv + λ·|fake − x|
5

Generate

At test time, pick a structure map from the class pool (or a small pose bank) and render. Later: swap the generator for a diffusion U-Net with the same condition.

But do not jump to Step 5 across 50 epochs yet — validate first ↓

Why validate before a full run

A full 50-epoch run is ~1.5–2.5 hours on your RTX 5060. Most idea-killing bugs (wrong conditioning wiring, a condition that carries no information, a model with too little capacity) show up in minutes if you test deliberately. The goal: fail in 10 minutes, not in 2 hours.

The validation ladder — cheapest test first

Climb in order. If a rung fails, stop and fix — don't pay for the next rung.

① Eyeball the conditioning maps~2 min · CPU

Plot 20 random (image, structure_map) pairs. Can you tell the letter from the map alone? If you can't, the network can't either.

Pass: the hand contour/shape is clearly visible and differs between classes.
Fail: maps are blank/noisy → tune Canny thresholds or use silhouette+distance only.
② Condition coverage check~1 min

Confirm the headline win over MediaPipe: every image yields a non-empty map.

cov = np.mean([m.any() for m in maps])
print(cov)   # expect ~1.00  (vs 0.27 for MediaPipe)
Pass: coverage ≈ 100%.
② Overfit a single batch~3 min · GPU

The classic wiring test. Train on one batch of 8–16 images for a few hundred steps with the adversarial term off (pure paired L1). A correct model will memorize them almost perfectly.

for step in range(300):
    loss = train_recon_step(fixed_batch)   # L1 only, condition→image
# plot generated vs real for the fixed batch
Pass: outputs converge to the exact training images → loss is wired right, model has capacity, condition is being used.
Fail: stays blurry → bug in conditioning path or target alignment. Fix before anything else.
③ The correspondence test (the one the old design failed)~3 min

Feed a real image's structure map and check the output reproduces that pose, not a class average.

Pass: changing the input map visibly changes the output hand pose → conditioning truly drives generation.
Fail: output ignores the map → the condition isn't connected deeply enough (inject it at multiple layers).
④ Tiny-subset run (2–3 classes)~10–15 min · GPU

Train the full pix2pix loss on ~600 images from 2–3 visually distinct letters for ~5 epochs. This is your first real "does the idea hold" signal.

Pass: samples are recognizable as the right letters and respond to the condition.
Fail: mode collapse / noise → adjust λ, learning rates, or condition injection before scaling.
⑤ Cheap quantitative probe~10 min

Train a tiny classifier on real images of those classes, then test it on your generated ones (the "GAN-test" / downstream-recognition metric). One number that actually means something.

Pass: the classifier labels generated samples well above chance (e.g. >70% on 3 classes) → the structure is being rendered faithfully.

Only after rungs ①–⑤ pass do you launch the full 50-epoch, 32-class run.

Total cost of the whole ladder: ~30–40 minutes vs a ~2-hour full run you'd otherwise risk on an unproven idea.

Pre-full-training checklist

✅ Structure maps are visually informative & cover 100% of images

✅ Model can overfit a single batch (loss + conditioning wired correctly)

✅ Output changes when the input condition changes (correspondence holds)

✅ 2–3 class subset produces recognizable, condition-driven hands

✅ A small classifier recognizes generated samples above chance

✅ Mixed precision / tf.data reused from src/ (fast from day one)

Tick all six → launch the full run with confidence. Miss one → you just saved yourself a wasted run.
