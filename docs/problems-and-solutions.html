Problems & How to Solve Them

Critical

① Structural losses regress toward the class average

both models · train step

The generated image comes from random noise z, but the pixel target is an unrelated real image of the same class. There is no correspondence between z and the target, so minimizing the L1 distance collapses toward the class mean. With λpix=5 this term dominates the adversarial loss → softer, less diverse hands.

Problem

fake = G([noise, label])      # random z
g_pix = mean(|fake - target|) # target = unrelated real
g_loss = g_adv + 5.0*g_pix    # pixel dominates

Solution

Replace pixel-L1-to-unaligned-target with feature matching (match the mean of D's intermediate features between the real and fake batch) — the standard collapse-resistant structural loss — or drop the term and lower λ.

g_fm = mean(|mean(D_feat(real)) -
            mean(D_feat(fake))|)
g_loss = g_adv + λ_fm * g_fm

Best fix: make structure a per-image condition (next page's pix2pix / diffusion idea) so the target and the input come from the same image — correspondence restored, no regress-to-mean.

Critical

② The MediaPipe landmark branch is weak (Model B)

Model B · regressor + landmark loss

Three compounding issues make the landmark signal noisy and biased:

Problem

a. The regressor is trained only on the 27% MediaPipe detected → unreliable on the rest.

b. The z-coordinate is destroyed: sigmoid→[0,1] + clip(0,1), but MediaPipe depth is often negative → 21 of 63 values zeroed.

c. Same misalignment as ①: loss compares unaligned fake vs real landmarks.

Solution

a. Use landmarks as a conditioning input, supervising only the detectable subset, not a loss over everything.

b. Drop z (use 42 = 21×x,y) or normalize z separately instead of clipping.

lm = lm.reshape(21,3)[:, :2]  # keep x,y only
lm = lm.flatten()             # 42 dims, no z clip

c. Pair landmarks with their own image (condition), not a random one.

High

③ Only ~27% of hands are detected

Model B · MediaPipe extraction

MediaPipe is trained on natural RGB hands in a scene; ArASL is tightly-cropped grayscale. The colorization cascade helps but the ceiling is low. This is a data mismatch, not a tuning bug.

Problem

73% of images produce zero landmarks → most of the dataset gives no structural signal, and the subset that works is biased toward "easy" poses.

Solution

Best: skip MediaPipe and use a structure map you can compute for 100% of images — an edge / silhouette / distance-transform from the grayscale hand. Always available, encodes sign shape directly.

edge = cv2.Canny(gray, 60, 160)   # 100% coverage

Alternative: newer MediaPipe Tasks HandLandmarker, or accept 27% and only condition on that subset.

Medium

④ The "diversity penalty" does not measure diversity

Model B · train step

Problem

It computes variance within a single image (= contrast), not variance across the batch, so it does nothing to prevent mode collapse and just rewards high contrast.

div = -0.05*var(flat, axis=1)  # within-image

Solution

Measure spread between samples of the same label, or use a proper mode-seeking term (maximize image distance when z differs).

div = -λ * mean(|G(z1)-G(z2)|)
            / (|z1-z2|)       # mode-seeking
Medium

⑤ Some evaluation metrics don't measure what they claim

both models · eval suite

Problem

SSIM / LPIPS are paired metrics, used here between a random fake and an arbitrary real of the class → near-meaningless (high SSIM would even signal collapse).

FID at ~2k samples with 2048-dim features is statistically biased (inflated, unstable).

PKLE relies on the broken 27% detector → computed on a few lucky pairs.

Solution

Use distributional metrics: keep FID but with more samples (or use KID, less biased at small N). Drop per-image SSIM/LPIPS-to-arbitrary-real.

Add the most meaningful metric for signs: downstream recognition — train a classifier on real, test on generated (and vice-versa).
