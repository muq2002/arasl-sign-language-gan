Overview

Both models share the same generator/discriminator backbone and training schedule. The only difference is how structure is supervised.

Model A — Pure cGAN

Adversarial loss + a pixel structural loss that pulls generated images toward real images of the same class. No hand-pose knowledge.

noise z → image label condition pixel L1 target

Model B — cGAN + MediaPipe

Everything in Model A plus a hand-landmark signal: MediaPipe extracts 21 joints, a frozen CNN regressor turns that into a differentiable landmark loss guiding hand geometry.

+ MediaPipe 21 joints + landmark regressor + landmark loss
Generator path Discriminator path Landmark path (Model B only) Loss weighting (λ)

Shared Backbone

Steps 1–3 are identical in both notebooks.

Data pipeline

54K PNGs (grayscale)→ resize 128×128→ normalize to [-1, 1]→ LabelEncoder → 32 classes→ shuffle (seed 42)

Generator (G)

A class-conditional up-sampler. Noise + label embedding are fused, then transposed-convolutions grow the spatial size 4→8→16→32→64→128. A SAGAN self-attention block at 32×32 captures global hand structure.

z (128)+ label emb (128)→ Dense 4×4×512→ ×5 Conv2DTranspose→ SelfAttention @32²→ tanh → 128×128×1

Discriminator (D)

Spectral-normalized conv stack. The class label is projected to a full 128×128 map and concatenated to the image, so D judges "is this a real and correctly-labelled hand?"

image 128²+ label map 128²→ SN Conv 128→64→32→16→8→4→ Flatten → Dense(1)→ real / fake logit
Stability tricks (both models): asymmetric learning rates (G=2e-4, D=1e-4), label smoothing (0.9), spectral norm on D, γ=0 self-attention init, adaptive G:D update ratio, and a two-phase pixel target (class-average → random real member).

Model A — Step by Step

Pure conditional GAN with a pixel structural loss. notebooks/model_A_cgan_128_no_mediapipe.ipynb

1

Load & preprocess

Read 54K grayscale signs, resize to 128², scale to [-1,1], encode the 32 class labels, shuffle.

2

Build class prototypes

Average all images of each class → a "prototype" used as the early structural target.

class_image_prototypes[c] = mean(images of class c)
3

Build G and D

Same backbone described above. ~ tens of millions of params each.

4

Train the Discriminator

One D step per batch: push real (smoothed to 0.9) up, push generated down.

5

Train the Generator (×2)

Adversarial loss (fool D) plus pixel L1 toward the target image, weighted by λ.

G = g_adv + λ · mean(|fake − target|)
6

Two-phase target + λ warm-up

Epochs 1–10: target = class prototype, λ ramps 0.5→5.0. Epoch 11+: target = a random real image of the class.

7

Adaptive balance + checkpoint

If G/D ratio drifts high, run extra G updates. Save weights, history, and sample grids every epoch.

Model B — Step by Step

Adds a hand-landmark signal on top of Model A. notebooks/model_B_cgan_128_mediapipe.ipynb

1

Load & preprocess

Identical to Model A.

2

Colorize grayscale for MediaPipe

MediaPipe expects skin-toned RGB. Each gray image is CLAHE-enhanced, padded, upscaled to 256px, and mapped to a synthetic skin tone before detection.

_gray_to_clahe_skintone()+ padding 15%
3

Extract 21 hand joints (4-strategy cascade)

Try CLAHE+skin@256 → @320 → CLAHE-only@0.15 → raw fallback. Returns 63 numbers (21 joints × x,y,z) or zeros on failure. Results are cached to disk.

extract_landmarks() → (63,)
4

Train a landmark regressor, then FREEZE it

A small CNN learns image → 63 coords from the images MediaPipe succeeded on. Once trained it is frozen and reused as a differentiable stand-in for MediaPipe.

build_landmark_regressor()trainable = False
5

Build per-class landmark prototypes

Average the valid landmarks per class (only classes with ≥50 detections are "gated" on).

6

Train D — same as Model A

No change to the discriminator.

7

Train G with THREE structural signals

Adversarial + pixel L1 + landmark MSE. The frozen regressor scores the fake image and the error flows back into G — so hand geometry guides generation, not just pixels.

G = g_adv + λpix·pixel + λlm·landmark + diversity
8

Two separate λ schedules

Pixel λ ramps 0.5→5.0 over epochs 1–10. Landmark λ stays 0 until epoch 15, then ramps to 2.0 — so geometry phases in only after pixels stabilize.

9

Checkpoint + sample — same as Model A

Honest Engineering Notes

Things worth knowing before trusting the results — kept here so the design is transparent.

~27% landmark detection. MediaPipe is trained on natural RGB hands in a scene; ArASL is tightly cropped grayscale. The colorization cascade helps but the ceiling is low. This is a data mismatch, not a tuning bug.
Structural losses regress toward the class mean. The generated image comes from random noise z, but the pixel/landmark target is an unrelated real image of the same class — there is no correspondence between z and the target. The minimizer is therefore the class average, which competes with sharpness and diversity (with λ_pix=5 the pixel term dominates the adversarial term).
The landmark branch is weak. The regressor is trained on only the detectable 27%, its z-coordinate is clipped to [0,1] (MediaPipe depth is often negative), and the loss compares unaligned samples — so even at 100% detection it would still push toward mean poses.
Metric caveats. SSIM/LPIPS are paired metrics used here on unpaired generations; FID at ~2k samples with 2048-dim features is statistically biased. A downstream recognition test (train on real, test on generated) would be more meaningful.
Recommended direction. Use structure (landmarks or always-available edge/silhouette maps) as a per-image conditioning input rather than a loss against unaligned targets — i.e. pix2pix / landmark-conditioned diffusion. That restores correspondence and removes the regress-to-mean problem.
