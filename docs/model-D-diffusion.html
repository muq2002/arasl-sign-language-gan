Model D — Structure-Conditioned Diffusion

The idea in one paragraph

Models A/B/C are GANs — a generator produces an image in a single shot and a discriminator judges it. Model D learns to denoise instead. We take a real image, add Gaussian noise on a schedule, and train a time-conditioned U-Net to predict the noise. The per-image structure map (Canny + silhouette + distance transform — the same signal that made Model C win) is concatenated to the noisy image so the network always knows the exact hand shape to render. To generate, we start from pure noise and run the denoiser backwards over a few DDIM steps.

structure map+ noisy x_t+ timestep t, class y→ U-Net → ε̂→ DDIM step × N→ sample

Forward process — add noise on a schedule

A cosine β-schedule defines how much noise to add at each timestep t. Sampling a random t per image and corrupting it gives the training target: the noise ε.

ᾱ_t = ∏(1 − β_i) # cosine schedule, t ∈ [0, 1000) x_t = √ᾱ_t · x₀ + √(1 − ᾱ_t) · ε # ε ~ N(0, I) is what the U-Net must predict

The conditioned U-Net (ControlNet-lite)

1

Concatenate the structure map at the input

Input is [x_t (1ch) ‖ structure (3ch)]. The map tells the network the exact silhouette and edges to reconstruct — full ControlNet is overkill at this resolution.

Concatenate([img, cond]) → 4ch
2

Inject timestep + class as a shared embedding

A sinusoidal timestep embedding plus a class Embedding (with one extra null row for CFG) are summed and added at every conv block.

temb(t) + Embedding(y)
3

Encoder–bottleneck–decoder with skip connections

Standard U-Net (GroupNorm + swish). Predicts the noise ε̂; the loss is a plain MSE.

loss = ‖ε̂ − ε‖²

Classifier-free guidance — the accuracy dial

During training we drop the class label ~10% of the time (replace it with the null class), so the same network learns both a conditional and an unconditional denoiser. At sampling we extrapolate between them with a guidance scale w.

train: y_in = y, but with prob 0.1 → ∅ # label dropout sample: ε = ε(∅) + w · (ε(y) − ε(∅)) # w=0 unconditional · w↑ stronger class adherence
The theory: higher w should make samples adhere more strongly to the class (higher recognition) at the cost of diversity. The notebook and harness sweep w and report recognition at each.
What the reduced run actually showed: guidance did not help — recognition went 0.819 → 0.788 → 0.769 as w rose (≈1 → 3 → 5), and diversity barely moved (~0.25). Model D landed at 0.82, below Model C's 0.95. The most likely cause is undertraining (10 epochs / 30 DDIM steps is far less than diffusion needs; the GANs converge fast at this tiny scale). Diffusion did train stably. Treat as directional — a fair comparison needs more epochs / steps, ideally on GPU. (w=0 ≡ w=1 here because the sampler shortcuts w=0 to the conditional path.)

Sampling with DDIM

Deterministic DDIM lets us denoise in ~30–50 steps instead of 1000, which keeps generation tractable (important on CPU). Each step predicts ε (conditional + unconditional for CFG), estimates x₀, and renoises to the next timestep.

x = noise
for t in linspace(T-1, 0, DDIM_STEPS):
    eps = eps(x, structure, t, y_null) + w*(eps(x, structure, t, y) - eps(x, structure, t, y_null))
    x0  = clip((x - sqrt(1-ab_t)*eps) / sqrt(ab_t), -1, 1)
    x   = sqrt(ab_next)*x0 + sqrt(1-ab_next)*eps
